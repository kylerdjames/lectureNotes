Global variables and global invariants.

- Multiple functions might modify the same global variables



- GLOBAL INVARIANTS specify constraints about global variables
    --> global invariants  must be true initially

    --> global invariants become both a precondition and a postcondition
    for all functions

    --> functions that modify global variables must:
        - not allow parameters that could violate global invariants
        - describe (as a postcondition) how global variables change




Example in practice:

public class Person {
    private String name;
    private int age;

    public Person(String n) {
        name = n;
        age = 0;
    }

    public void setAge(int a) {
        age = a;
    }
}

--> good global invariant? (would it be true initially?)

--> setAge precondition?

--> setAge postcondition?

Termination - whether a program will finish on all inputs


Partial Correctness:
    IF the precondition is satisfied
    THEN we guarantee the postcondition, but only IF 
        the program terminates


Total Correctness:
    IF the precondition is satisfied
    THEN we guarantee the postcondition AND termination


We need to show termination to achieve total correctness



////////////////////////////////////////

Termination with a loop.
    - create measure of how much work is left
        (i.e., how many iterations are left)
        --> should be a nonnegative integer

    - show that the measure DECREASES with each iteration

    - show that if there is no more work to do, then the loop
        has finished
