Problem 3 on HW 11 (reverse sequence):

function contract postcondition:
for all positions k, seq(k) == In(seq)(seq.size-k-1)

loop invariant:
size of seq doesn't change
bound i (starts at 0, <= seq.size/2)
1) which ones have changed at the beginning 
    first i positions HAVE swapped (use the postcondition format)
    last i positions HAVE swapped (use the postcondition format)
    middle have NOT changed (from i to seq.size-i)


def reverseSeq(seq: ZS): Unit = {
  var i: Z = 0
  while (i < seq.size/2) {
  var temp: Z = seq(i)
    seq(i) = seq(seq.size-i-1)
    seq(seq.size-i-1) = temp
    i = i + 1
  }
}

writing a class to represent a clock (time of day)
    fields: hours, minutes, seconds, am/pm 

    minutes should ALWAYS be between 0-59 (this is a global invariant)

- GLOBAL INVARIANTS specify constraints about global variables
    --> global invariants  must be true initially

    --> global invariants become both a precondition and a postcondition
    for all functions

    --> functions that modify global variables must:
        - not allow parameters that could violate global invariants
        - describe (as a postcondition) how global variables change




Example in practice:

Problem 6 on HW 11: Fan specification

Fan speeds: 0-5
Fan can be on (speeds 1-5) or off (speed 0)
Fan can optionally rotate (but ONLY at speeds 3-5)

Think of it as having two controls:
    1) speed knob (becomes on when 1-5, off when 0)
        --> can only change one speed level at a time

    2) rotation lever

--> since rotation is a DIFFERENT control, changing
    speed can't change rotation


Initially:
    fan off
    speed 0
    rotate off

Global invariants:
    curSpeed >= 0,
    curSpeed <= 5,
    fanOn == true -> curSpeed > 0,
    fanOn == false -> curSpeed == 0,
    rotateOn == true -> curSpeed >= 3


Functions:
    setSpeed(speed: Z)
    turnOn()
    turnOff()
    makeRotate()
    stopRotate()



    Fan on, speed 3, rotate on. Can we call setSpeed(2)?
        NO. This should not be allowed since the speed would be 2 while rotating.
        need to require if speed parameter is < 3 then the fan is NOT rotating

    Fan off, speed 0, rotate off. Can we call setSpeed(1)?
        yes
        and the fan should be marked on

    Fan on, speed 2, rotate off. Can we call setSpeed(0)?
        NO. can only change the speed by 1 in each direction.

    Fan on, speed 3, rotate on. Can we call setSpeed(4)?
        yes

    Fan on, speed 3, rotate on. Can we call makeRotate()?
        NO. Fan is already rotating.

    Fan on, speed 5, rotate off. Can we call setSpeed(5)?
        NO. This wouldn't be a change.

    Fan on, speed 5, rotate off. Can we call setSpeed(6)?
        NO. This would make the current speed 6 (and violate the   
            global invariant)

    Fan on, speed 1, rotate off. Can we call setSpeed(0)?
        yes.
        and would be marked off.



setSpeed(speed: Z) preconditions?
    Require: speed parameter needs to be between 0-5
        speed parameter should be 1 bigger or 1 smaller than current speed
        if speed parameter < 3 then rotate needs to be off


    postconditions?
        describe that the current speed changes to be that parameter speed

Termination - whether a program will finish on all inputs


Partial Correctness:
    IF the precondition is satisfied
    THEN we guarantee the postcondition, but only IF 
        the program terminates


Total Correctness:
    IF the precondition is satisfied
    THEN we guarantee the postcondition AND termination


We need to show termination to achieve total correctness
