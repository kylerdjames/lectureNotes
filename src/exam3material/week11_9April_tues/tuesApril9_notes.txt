#5: Use proof by contradiction to prove that the sum of a rational and
irrational number is irrational.

How to write as an IF/THEN?
    IF we have a rational number and an irrational number
    THEN their sum is irrational

Rational definition?
    A number r is considered rational if r = p/q, where p and q 
    are integers and q !=0.

    A number is irrational if it is not rational.


How to start the proof? 

Assume we have a rational number r and an irrational number i.
Assume, for the sake of contradiction, that i+r is rational.

Since r is rational, r = p/q where p, q are integers and q != 0.
Since i+r is rational, i+r = a/b, where a, b are integers and b != 0.

We have:
    i = (i+r) - r
     = a/b - p/q 
     (put it over a common denominator)
     (show the common denominator is not 0)

After doing that, you have a contradiction -- i would meet the definition
of rational, but we assumed it was irrational.

(Goal: contradiction)

Preconditions: 
    assumptions we need to make before some section of code


Postconditions: 
    things we want to prove our program does

/////

assume statements
    - lets us make assumptions about user input
    - we can pull these assumptions into a proof



Example:
    var input: Z = Z.prompt("Enter a positive number: ")
    assume(input > 0)

/////

assert statements
    - what we want to prove is true after some code
        (postconditions)

    - must prove EXACTLY that statement before we assert it



assert(some claim)


How are Logika asserts different from C# (and other languages)?

Logika: assert is satisfied if we have PROVED what it claims

Other languages (like C#): asserts call the code, are satisfied
    if it returns what we are asserting

Another justification: Subst









/////////////////////////////

Subst_<(m, n)



Deduce(
    ...
    m ( LHS_M == RHS_M ),      //must be an equivalence
    ...
    n ( claim1 ),
    ...
    j ( claim2 )              by Subst_<(m, n)
)

claim2 rewrites claim1 by substituting all occurrences of
    LHS_M with RHS_M


    (the < points at the value we are REPLACING)



Example:

Deduce(
    ...
    m ( x + 1 == y - 4 ),
    ...
    n ( x*(x + 1) == (x + 1) + y ),
    ...
    j ( x*(y-4) == (y-4) + y )     by Subst_<(m, n)
)





/////////////////////////////

Subst_>(,m n)

Deduce(
    ...
    m ( LHS_M == RHS_M ),      //must be an equivalence
    ...
    n ( claim1 ),
    ...
    j ( claim2 )             by Subst_>(m, n)
)

claim2 rewrites claim1 by substituting all occurrences of
    RHS_M with LHS_M



Example:

Deduce(
    ...
    m ( x + 1 == y )
    ...
    n ( y*(x + 1) == (x + 1) + y )
    ...
    j ( (x+1)*(x + 1) == (x + 1) + x + 1 ) by Subst_>(m, n)
)
